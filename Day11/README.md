Lab: Day 11 - Troubleshooting Static Routes (Cisco Packet Tracer)

Objective: Identify and fix exactly one misconfiguration on each router (R1 and R2) that is preventing PC1 and PC2 from reaching each other across the four networks: 192.168.1.0/24, 192.168.12.0/24, 192.168.13.0/24, and 192.168.3.0/24.
Task: Use troubleshooting tools such as ping and show ip route to diagnose broken static routes on R1 and R2, then correct the faulty route statements — as shown by the initial 50% packet loss on PC1 before the fix, and full 0% loss after.
Verification: The lab is successfully completed when PC1 can ping PC2 (192.168.3.1) with 0% packet loss across all 4 replies, confirming that both misconfigured static routes have been correctly identified and resolved.
